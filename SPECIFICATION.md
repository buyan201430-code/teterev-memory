# Спецификация цифрового контура проекта «Михаил Тетерев»

> Цифровой узел памяти, архива, сбора свидетельств и продвижения книги / аудиокниги / документального цикла о тренере по боксу из Благовещенска (Амурская область).

**Ключевая ось:** Тренер может вырастить сильного человека даже без системы, но если система не даёт условий, ученик вынужден уходить туда, где есть поддержка.

**Формула проекта:** Ученик уходит не от тренера, а от отсутствия условий.

## 1. Сайт

### 1.1. Цель

Сайт — не визитка, а **рабочая система сбора памяти**. Цифровой контур (сайт + Telegram + архив в Git + формы + медиа) работает как единый механизм:

```
человек → форма / канал → карточка архива → верификация → публикация → глава книги / ролик / подкаст
```

**Что делает система:**

- принимает свидетельства, фото, газеты, имена;
- превращает их в структурированные карточки с ID и статусом достоверности;
- показывает пробелы («кого ищем», «что не подтверждено»);
- публикует только то, на что есть разрешение;
- кормит книгу, YouTube и подкаст проверенными материалами.

**Роли сайта в контуре:**

- публичная витрина архива (газеты, фото, цитаты);
- страница «Ищем людей» — активный поиск свидетелей;
- точка входа для учеников, родственников, журналистов;
- формы с юридическим блоком разрешений;
- инструмент верификации и фиксации пробелов;
- постоянная ссылка для прессы и спортивного сообщества.

### 1.2. Главная идея

**«Школа силы без системы»** — документальная история тренера и его учеников на фоне города, который не смог удержать тех, кого вырастил.

Первый экран должен сразу отвечать на три вопроса:

1. Кто такой Михаил Тетерев?
2. Почему эта история важна сейчас?
3. Как я могу помочь?

### 1.7. Визуальный стиль и тон

См. раздел [8. Дизайн](#8-дизайн).

**Тон текста:**

- документальный, без пафоса и глянца;

- пометки достоверности: *подтверждено* / *со слов* / *требует проверки*.

### 1.8. Мобильная версия

- одноколоночная вёрстка;
- навигация — бургер + липкая нижняя панель: Архив | Прислать | Telegram;
- карточки архива — свайп-лента или вертикальный список;
- газетные сканы — pinch-to-zoom, полноэкранный просмотр;
- формы — крупные поля, минимум обязательных;
- шрифт body ≥ 16px, контраст WCAG AA;
- фото не обрезать агрессивно — сохранять архивный кадр.

## 2. Архив

### 2.1. Принципы

1. **Первоисточник ≠ обработка** — хранятся отдельно, связаны ссылкой.
2. **Каждый объект имеет ID** — `NP-1987-042`, `STU-ivanov-ii`, `EVT-1991-championship`.
3. **Статус достоверности** обязателен на каждой карточке.
4. **Git + облако** — двойное резервирование (вариант C).
5. **Не удалять** — помечать `deprecated`, `superseded_by`.

### 2.2. Хранение газетных сканов

```
/archive/newspapers/
  /raw/                    # оригинальные сканы, без правок
    NP-1987-042_raw.tiff
  /processed/              # обрезка, выравнивание, контраст
    NP-1987-042_proc.jpg
  /thumbnails/
    NP-1987-042_thumb.jpg
```

- **Формат raw:** TIFF или PNG, 300+ dpi.
- **Формат web:** JPEG/WebP, max 2000px по длинной стороне.
- **Именование:** `NP-{год}-{номер}_{raw|proc|thumb}.{ext}`
- **Метаданные:** sidecar-файл `NP-1987-042.yaml` или `.md` карточка.

### 2.3. Хранение расшифровок

```
/archive/newspapers/
  /transcripts/
    NP-1987-042_transcript.md    # полный текст
    NP-1987-042_summary.md       # краткая выжимка для сайта
```

Расшифровка — отдельный файл. В карточке статьи — ссылка на оба. OCR-версия помечается `source: ocr`, ручная — `source: manual`.

### 2.4. Первоисточник vs обработка

| Тип | Папка | Поле `record_type` |
|-----|-------|-------------------|
| Скан газеты | `/raw/` | `primary_source` |
| Расшифровка | `/transcripts/` | `derived_text` |
| Выжимка | `/transcripts/` | `summary` |
| Цитата на сайте | в CMS | `public_excerpt` |
| Комментарий редактора | в карточке | `editorial_note` |
| Интерпретация для книги | `/book/drafts/` | `interpretation` |

### 2.5. Статусы достоверности

| Статус | Код | Когда |
|--------|-----|-------|
| Подтверждено | `verified` | 2+ независимых источника или документ |
| Со слов | `oral` | интервью, воспоминание без документа |
| Требует проверки | `needs_review` | одна ссылка, противоречие, неполные данные |
| Сомнительно | `disputed` | конфликт версий |
| Не подтверждено | `unverified` | догадка, слух |

Отображение на сайте: бейдж цветом (см. дизайн).

### 2.6. Список имён «на второй круг»

Файл [`archive/people_second_pass.md`](archive/people_second_pass.md) — рабочий реестр людей, которых нужно найти, подтвердить или опросить повторно.

Обновлять при: новой газетной статье, новом воспоминании, закрытии пробела (`gaps.md` → resolved).

Файл [`archive/gaps.md`](archive/gaps.md) — реестр того, чего не хватает в архиве: неподтверждённые факты, пропущенные даты, разрывы в биографиях.

Каждый пробел получает ID `GAP-NNN` и ссылки на связанные карточки.

### 2.9. Карточка факта (FACT)

Атомарная единица достоверности. Один проверяемый факт — одна карточка `FACT-...`.

Примеры: «Жданов назвал Тетерева первым тренером» (источник NP-...), «Секция работала без бюджета на поездки» (источник INT-..., oral).

Факты связывают материалы между собой и питают хронологию, карточки учеников и главы книги. Несколько FACT могут подтверждать или противоречить друг другу — статус `disputed` фиксируется явно.

### 2.10. Карточка интервью (INT)

Фиксирует: кто, когда, в каком формате, где лежит запись, где расшифровка, какие FACT извлечены, какие разрешения получены (см. раздел 11).

## 3. Telegram-канал

### 3.1. Зачем нужен

- **Быстрый охват** — родственники и ученики чаще в Telegram, чем на сайте.
- **Регулярный ритм** — каждая находка сразу видна подписчикам.
- **Обратная связь** — комментарии, пересылки, личные сообщения.
- **Доверие** — живой процесс «мы ищем и находим», а не готовый монумент.
- **Трафик на сайт** — короткие посты → ссылка на полную карточку архива.

4. Благодарность: каждый приславший — упоминание (с согласия).
5. Не публиковать личные данные без разрешения.

## 4. YouTube

### 4.1. Зачем нужен канал

- **Голос и лицо** — доверие через живое интервью.
- **Визуал архива** — газетные сканы, фото зала, карта города.
- **Долгий хвост** — видео находят через поиск «бокс Благовещенск», «Тетерев».
- **Материал для книги** — расшифровки интервью → главы.
- **Трейлер книги** — перед выходом печатного издания.

### 4.2. Форматы

| Формат | Длина | Назначение |
|--------|-------|------------|
| Shorts | 30–60 сек | Цитата, факт, фрагмент газеты |
| Находка недели | 3–5 мин | Одна статья / одно имя |
| Интервью | 20–45 мин | С учеником, родственником |
| Документальный выпуск | 40–60 мин | Смонтированная глава |
| Трейлер проекта | 2–3 мин | Анонс книги / серии |
| За кадром | 5–10 мин | Как ищем архив |

### 4.3. Короткие видео (Shorts)

- Газетная строка крупным планом + закадровый голос.
- «Знал ли ты, что...» — один факт.
- Цитата ученика на фоне архивного фото.
- «Ищем этого человека» — карточка ученика.
- Фрагмент интервью — 30 сек самого сильного момента.

### 4.4. Длинные видео

- Полное интервью с расшифровкой.
- Разбор газетной статьи: скан → текст → контекст.
- «Те, кто ушёл» — судьба одного ученика.
- «Тренер без системы» — разговор с коллегами-тренерами.
- Документальный выпуск к выходу главы книги.

### 4.5. Использование газетных сканов

1. **Ken Burns** — медленное движение по скану.
2. **Подсветка строки** — выделение цитаты.
3. **Split screen** — скан слева, расшифровка справа.
4. **OCR-анимация** — текст «проявляется» поверх скана.
5. Всегда указывать: газета, дата, источник в углу кадра.

### 4.6. Использование интервью

- Три камеры не нужны — достаточно статичного кадра + B-roll (фото, зал, город).
- Начинать с сильной цитаты (cold open).
- Титры: имя, годы занятий, роль.
- Архивные вставки по ходу рассказа.
- Финал — призыв: «Если вы помните...» + ссылка.

### 4.7. Трейлер проекта (2–3 мин)

1. **0:00–0:15** — архивное фото, звук зала, голос: «Его называли первым тренером».
2. **0:15–0:45** — 3–4 быстрых газетных фрагмента, имена учеников.
3. **0:45–1:15** — формула проекта, город, карта.
4. **1:15–2:00** — фрагменты интервью (если есть) или закадровый текст.
5. **2:00–2:30** — «Мы ищем свидетелей. Книга. Подкаст. Архив.» + URL.

### 4.8. Структура первого видео (10–15 мин)

**«Кто такой Михаил Тетерев — и зачем мы ищем его историю»**

1. Хук — цитата из газеты (30 сек).
2. Автор проекта — кто мы, почему взялись (1 мин).
3. Благовещенск, бокс, 80–90-е — контекст (2 мин).
4. Что уже найдено — 2–3 газетные находки (3 мин).
5. Ключевая ось — «ученик уходит не от тренера...» (2 мин).
6. Чего не хватает — имена, фото, свидетели (2 мин).
7. Как помочь — сайт, формы, Telegram (1 мин).
8. Финал — «Это только начало» + титры.

### 4.9. Структура документального выпуска (40–60 мин)

1. Cold open — сильная сцена / цитата.
2. Титры + музыка (сдержанная, без героического оркестра).
3. Глава 1 — контекст (город, система, зал).
4. Глава 2 — тренер (как работал, метод).
5. Глава 3 — ученики (2–3 судьбы).
6. Глава 4 — «те, кто ушёл».
7. Глава 5 — что осталось / память.
8. Финал — связь с книгой и архивом.
9. Credits + ссылки.

### 4.10. 10 тем первых роликов

1. Запуск проекта — кто такой Тетерев
2. Первая газетная находка — разбор статьи
3. «Первый и любимый тренер» — откуда эта формула
4. Благовещенск и бокс в 80–90-е — контекст
5. Ищем учеников — призыв + одно имя
6. Как устроен архив — процесс для зрителя
7. Спортзал — фото и воспоминания о месте
8. «Бокс как шахматы» — о методе тренера (по имеющимся источникам)
9. Shorts-серия: 5 цитат из газет
10. Интервью №1 (когда появится свидетель)

## 5. Аудиокнига / подкаст

### 5.1. Чем отличается аудиокнига от подкаста

| | Аудиокнига | Подкаст / аудиосериал |
|---|-----------|----------------------|
| Сценарий | Финальный текст, вычитанный | Живой, может меняться |
| Структура | Главы книги | Эпизоды по темам |
| Длина | 15–25 мин/глава | 20–40 мин/выпуск |
| Гости | Редко | Часто |
| Цель | Прочитать книгу | Искать и собирать |
| Когда | После черновика книги | **С самого начала** |

### 5.2. Формат для старта

**Аудиосериал «Школа силы»** — 20–30 мин, 1 выпуск / 2 недели.

Почему не аудиокнига сразу: текст книги ещё пишется; подкаст-формат позволяет публиковать находки, интервью и черновые главы; каждый выпуск — призыв к свидетелям.

### 5.3. Структура аудиосериала

- **Сезон 1 (6–8 выпусков):** поиск, архив, первые свидетели.
- **Сезон 2:** «Те, кто ушёл» — судьбы учеников.
- **Сезон 3:** черновые главы книги в аудио.
- **Финал:** полная аудиокнига — перезапись вычитанного текста.

### 5.4. Платформы

**Старт (бесплатно):**
- Anchor / Spotify for Podcasters
- Яндекс Музыка (Подкасты) — через RSS
- VK Подкасты
- Telegram — аудиофайлы + ссылка

**Позже:**
- Литрес (аудиокнига — после ISBN и договора)
- Storytel, СберКниги
- YouTube — полная версия с статичным кадром

### 5.5. Выход на Литрес / Яндекс Музыку

1. Завершить текст книги, профессиональная запись (студия).
2. ISBN печатной / электронной книги.
3. Договор с Литрес (самостоятельно или через издательство).
4. Обложка, аннотация, демо-фрагмент 5 мин.
5. Яндекс Музыка — через дистрибьютора (TuneCore, DistroKid) или партнёра Литрес.

### 5.6. Структура первого аудиоэпизода (25 мин)

1. **Интро (1 мин)** — музыка, название сериала.
2. **Зачем мы здесь (3 мин)** — автор, проект, формула.
3. **Кто такой Тетерев (5 мин)** — известные факты, осторожно с непроверенным.
4. **Первая находка (7 мин)** — читаем газетный фрагмент, обсуждаем.
5. **Что ищем (4 мин)** — имена, фото, свидетели.
6. **Как помочь (2 мин)** — сайт, формы.
7. **Аутро (1 мин)** — анонс следующего выпуска.

### 5.7. 10 тем выпусков

1. Запуск — зачем «Школа силы»
2. «Первый и любимый тренер» — газетные свидетельства
3. Благовещенск — город и спорт
4. Тренер без системы — как это выглядело
5. Энтузиазм вместо бюджета — зал, снаряды, быт
6. Бокс как шахматы — метод Тетерева
7. Имя из архива — судьба одного ученика
8. Те, кто ушёл — почему и куда
9. Интервью со свидетелем
10. Итоги сезона — что нашли, что дальше

## 6. Книга

### 6.1. Возможное название

**«Школа силы»**

### 6.2. Подзаголовок

**«История тренера Михаила Тетерева и тех, кого Благовещенск не удержал»**

*Альтернативы:*
- «Как растят боксёров без системы — и почему ученики уходят не от тренера»
- «Первый тренер. Город, который отпустил»

### 6.3. Главная ось

Тренер может вырастить сильного человека даже без системы. Но когда система не даёт условий — ученик уходит не от тренера, а от отсутствия условий. Это история человека, школы и города.

### 6.4. Структура глав

| № | Глава | Содержание | Статус по материалам |
|---|-------|------------|---------------------|
| Пролог | — | Зачем эта книга, формула, метод сбора | можно писать |
| 1 | **Первый тренер** | Кто такой Тетерев; формулировка учеников из газет | **есть материалы** |
| 2 | **Школа силы** | Зал, атмосфера, первые ученики | частично |
| 3 | **Бокс как шахматы** | Метод, тактика, характер тренера | мало, нужны интервью |
| 4 | **Энтузиазм вместо бюджета** | Условия, снаряды, ремонт, волонтёрство | нужны свидетели |
| 5 | **Тренер без системы** | Спортивная бюрократия, поддержка, её отсутствие | нужны интервью |
| 6 | **Те, кто ушёл** | Судьбы учеников, переезды, причины | **есть зачатки в газетах** |
| 7 | **Город, который не удержал** | Благовещенск, регион, контекст 80–90–00-х | нужен контекст |
| 8 | Голоса | Коллаж цитат — ученики, родные, коллеги | по мере сбора |
| 9 | Что осталось | Память, след, современное состояние | — |
| Эпилог | — | Зачем помнить; архив; благодарности | — |

### 6.5. Какие материалы нужны

- Газетные статьи (сканы + расшифровки) ✅ частично есть
- Список учеников с годами и достижениями
- 10–15 глубоких интервью
- Фото: тренер, зал, соревнования, группы
- Документы: приказы, diplomas, программы соревнований
- Карта «кто куда уехал»
- Хронология 1970–2020 (уточняется)

### 6.6. Кого опросить

| Группа | Кого искать | Приоритет |
|--------|-------------|-----------|
| Ученики | все упомянутые в газетах | высокий |
| Ученики | уехавшие из города | высокий |
| Родственники | дети, внуки, супруги Тетерева | высокий |
| Коллеги-тренеры | из Благовещенска и области | средний |
| Спортивные функционеры | кто помнит условия | средний |
| Журналисты | авторы статей | средний |
| Соседи / друзья | бытовые воспоминания | низкий |

### 6.7. Вопросы ученикам

1. Когда вы впервые пришли к Тетереву? Сколько вам было лет?
2. Почему выбрали бокс? Почему именно к нему?
3. Каким вы запомнили тренера — характер, привычки, фразы?

| | |

- GitHub private repo — все `.md` карточки

```
variant: raw | proc | thumb | transcript
```

## 11. Юридический блок разрешений

Каждый материал от живого человека (воспоминание, фото, интервью) проходит через блок разрешений **до публикации**. Газеты и документы из публичного архива — отдельный режим (источник указан, статус `primary_source`).

### 11.1. Шесть разрешений

| # | Вопрос | Поле в форме / INT | Значения |
|---|--------|-------------------|----------|
| 1 | Можно публиковать на сайте? | `perm_website` | `yes` / `no` / `anonymous` |
| 2 | Можно использовать в книге? | `perm_book` | `yes` / `no` / `anonymous` / `with_edits` |
| 3 | Можно использовать в видео (YouTube)? | `perm_video` | `yes` / `no` / `voice_only` / `anonymous` |
| 4 | Можно использовать в аудио (подкаст / аудиокнига)? | `perm_audio` | `yes` / `no` / `anonymous` |
| 5 | Можно указывать имя отправителя? | `perm_name` | `full_name` / `first_name_only` / `anonymous` / `no` |
| 6 | Можно связываться повторно? | `perm_recontact` | `yes` / `no` / `via_messenger_only` |

### 11.2. Комбинации по умолчанию

| Тип материала | website | book | video | audio | name | recontact |
|---------------|---------|------|-------|-------|------|-----------|
| Только для архива | no | no | no | no | по выбору | yes |
| Открытая публикация | yes | yes | yes | yes | full_name | yes |
| Анонимное свидетельство | anonymous | anonymous | anonymous | anonymous | anonymous | no |
| Только книга | no | yes | no | no | по выбору | yes |

### 11.3. Где фиксировать

1. **Формы на сайте** — чекбоксы / radio (расширить Форму 1 и 2, см. ниже).
2. **Карточка INT-...** — блок `permissions:` в YAML frontmatter.
3. **Файл согласия** — `interviews/consents/{INT-id}_consent.md` или скан подписи.
4. **Переписка Telegram** — скрин / экспорт, ссылка в INT-карточке (дата, `@username`).

### 11.5. Правила публикации

1. **Без явного `yes`** — материал остаётся в закрытом архиве, на сайте не появляется.
2. **`anonymous`** — публикуется текст / фото без имени; в карточке имя хранится только у команды.
3. **`with_edits`** — перед книгой согласовать финальный фрагмент с автором.
4. **Фото третьих лиц** — нужно разрешение изображённых или обоснование (публичное событие, газета).
5. **Несовершеннолетние на фото** — публикация только с согласия родителей.
